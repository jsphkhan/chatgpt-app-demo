Building Apps for ChatGPT

The four building blocks and how they connect in this demo

Basic building blocks

A ChatGPT app built with the Apps SDK combines four pieces. Each has a distinct job; together they let the model call your backend and render a rich UI inside the chat.

BlockRole in this project
MCP Server server.js — exposes tools and UI resources over /mcp for ChatGPT to discover and call.
UI Widget/Web Component React app in src/, built to dist/index.html — the horoscope card ChatGPT renders in the chat.
Bridge @modelcontextprotocol/ext-apps — PostMessageTransport + useApp connect the widget iframe to ChatGPT via JSON-RPC over postMessage.
Sandboxed ChatGPT iframe ChatGPT hosts your widget HTML in an isolated iframe with CSP — your code cannot access the parent chat DOM; it receives tool results through the bridge.

High-level overview

MCP Server

DescriptionDiagram

The Model Context Protocol (MCP) server is the backend ChatGPT talks to. It advertises tools the model can call (with JSON Schema inputs/outputs) and resources (including your widget HTML).

In this demo, Express serves Streamable HTTP at /mcp. ChatGPT sends tools/call and resources/read requests; your handlers return structured results.

UI Widget

DescriptionStack in this demo

The UI widget is the visual layer inside ChatGPT — a card, form, or dashboard rendered in the sandboxed iframe after a tool runs.

You can build it with React (as here), Vue, Svelte, or vanilla JS. Vite bundles your app into dist/; the MCP server serves that HTML and assets when ChatGPT requests the registered resource URI.

The widget listens for tool events via useApp and updates state — e.g. skeleton on ontoolinput, horoscope card on ontoolresult. 

src/App.tsx          → React UI + useApp
src/main.tsx         → mount point
vite build           → dist/index.html + assets/
widget.js            → registerAppResource(...)
ChatGPT iframe       → renders dist/index.html

Bridge between ChatGPT & your MCP server

The widget does not call MCP server directly. ChatGPT sits in the middle: MCP tool calls on the server, and the Apps SDK bridge inside the iframe for UI updates.

Code — useApp in App.tsxDescription
 
import { useApp } from "@modelcontextprotocol/ext-apps/react";

const { isConnected, error } = useApp({
  appInfo: { name: "horoscope-widget", version: "0.1.0" },
  capabilities: {},
  onAppCreated: (app) => {
    app.ontoolinput = () => {
      setLoading(true);
      setHoroscope(null);
    };

    app.ontoolresult = (result) => {
      setLoading(false);
      const data = result.structuredContent;
      if (data?.sign && data.points?.length) {
        setHoroscope(data);
      }
    };

    app.ontoolcancelled = () => {
      setLoading(false);
    };
  },
});

useApp from @modelcontextprotocol/ext-apps/react is the widget-side bridge. It creates an MCP App instance, connects to ChatGPT via PostMessageTransport (under the hood), and exposes familiar event handlers — you never wire window.postMessage yourself.

Two channels, one flow:

  1. MCP — ChatGPT calls show_horoscope on server.js and receives structuredContent
  2. useApp events — ChatGPT notifies the iframe; your handlers react:
    • ontoolinput → show skeleton
    • ontoolresult → read structuredContent, render card
    • ontoolcancelled → clear loading state

Registering Tools with MCP Server

registerAppTool

CodeDescription
 
// tools/show-horoscope.tool.js
registerAppTool(
  server,
  "show_horoscope",
  {
    title: "Show horoscope",
    description: "Displays a personalized horoscope card…",
    inputSchema: showHoroscopeInputSchema,
    outputSchema: showHoroscopeOutputSchema,
    _meta: {
      ui: {
        resourceUri: widgetUri,
        csp: widgetCsp,
      },
    },
  },
  handleShowHoroscope
);

Registers an action ChatGPT's model can call during conversation. The description tells the model when and how to use the tool; inputSchema defines arguments (here, dob).

_meta.ui.resourceUri links this tool to the widget registered by registerAppResource. The handler returns text for the chat plus structuredContent for the UI.

Registering UI Resources with MCP Server

registerAppResource

CodeDescription
 
// widget.js
registerAppResource(
  server,
  "horoscope-widget",
  WIDGET_URI,
  { _meta: { ui: { csp } } },
  async () => ({
    contents: [
      {
        uri: WIDGET_URI,
        mimeType: RESOURCE_MIME_TYPE,
        text: widgetHtml,
        _meta: { ui: { csp } },
      },
    ],
  })
);

Publishes your built React HTML as an MCP resource ChatGPT can fetch. When a tool points to WIDGET_URI, ChatGPT calls this callback and receives dist/index.html (with asset URLs rewritten to your public server).

CSP in _meta.ui whitelists domains the sandboxed iframe may load scripts and styles from (your ngrok host in dev).
