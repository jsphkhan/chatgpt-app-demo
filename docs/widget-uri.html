WIDGET_URI

The ID that links your MCP tool to the React UI

WIDGET_URI is defined in config.js and used across the server to tell ChatGPT which UI to render when a tool runs.

What is WIDGET_URI?

It is the ID of your React UI in the MCP system — not a real web address you visit in a browser.

Code Description 
// config.js
export const WIDGET_URI = "ui://widget/horoscope.html";

The constant defined in config.js — the ID ChatGPT uses to look up your widget UI.

Part What it means
ui:// MCP Apps convention — signals "this is a UI resource", not a normal https:// URL.
widget/horoscope.html A label you choose. It does not need to match a real file path on disk. Your actual HTML lives in dist/index.html.

Why is it needed?

Description How it links tool → UI

ChatGPT needs two separate pieces:

  1. A tool — show_horoscope (what the model calls with the user's DOB)
  2. A UI resource — the React card (what gets rendered in the iframe)

WIDGET_URI is the string that links them together. When the tool runs, ChatGPT looks up the UI registered at this URI and renders it.

The tool's resourceUri must match the URI registered by registerAppResource. Both use the same WIDGET_URI constant from config.js.

Tool points to the URI

CodeDescription
 
// tools/show-horoscope.tool.js
_meta: {
  ui: {
    resourceUri: widgetUri,  // "ui://widget/horoscope.html"
    csp: widgetCsp,
  },
}
 Tells ChatGPT: "After calling this tool, render the UI registered at ui://widget/horoscope.html."

Server registers HTML at that URI

CodeDescription
 
// widget.js
registerAppResource(
  server,
  "horoscope-widget",
  WIDGET_URI,
  { _meta: { ui: { csp } } },
  async () => ({
    contents: [
      {
        uri: WIDGET_URI,
        mimeType: RESOURCE_MIME_TYPE,
        text: widgetHtml,
        _meta: { ui: { csp } },
      },
    ],
  })
);
 Registers your built React HTML at that URI. When ChatGPT requests ui://widget/horoscope.html, this callback returns dist/index.html content.

How it flows

Description Diagram
  1. User asks for horoscope in ChatGPT
  2. ChatGPT calls show_horoscope tool
  3. Tool metadata says: resourceUri = "ui://widget/horoscope.html"
  4. ChatGPT requests that resource from your MCP server
  5. widget.js returns the built HTML for that URI
  6. Horoscope card appears in the chat iframe

Where it's used

File Role
config.js Defines the constant
widget.js Registers the HTML at this URI
tools/show-horoscope.tool.js Points the tool to this URI via resourceUri
server.js Passes WIDGET_URI into the tool registration

Important: all files must use the same string. If they differ, ChatGPT won't know which UI to show.

Analogy

Concept Description
WIDGET_URI Like a function name in code — an identifier ChatGPT uses to look up your widget.
dist/index.html The actual file on disk — the real HTML content returned when ChatGPT asks for that URI.

One-line summary: WIDGET_URI is the name. dist/index.html is the content ChatGPT gets when it asks for that name.
