Running & Testing

1. Testing locally

  1. Start your MCP server 
    npm run build:start

    Builds the widget, then starts the server at http://localhost:3000/mcp

  2. Start MCP Inspector 
    npm run inspector

    Opens the inspector against your local MCP endpoint. Call show_horoscope with { "dob": "1990-08-15" } to verify the tool response.

2. Testing on ChatGPT

Run the MCP server in developer mode with a public HTTPS URL:

  • Enable Developer mode in ChatGPT: Settings → Apps & Connectors → Advanced settings
  • Build and expose your server with ngrok: 
    npm run start:ngrok
  • Copy the printed MCP URL (e.g. https://your-domain.ngrok.dev/mcp)
  • In ChatGPT: Settings → Connectors → Create → paste the URL and save
  • Open a new chat, add your connector from the + menu → More
  • Ask for your horoscope — ChatGPT will call show_horoscope and render the widget
  • After changing tools or server code, refresh the connector in Settings → Connectors

  • registerAppTool — registers the horoscope tool ChatGPT calls (tools/)
  • CSP & Security — domain allowlist so the sandboxed widget can load JS/CSS from your server (widget.js, config.js)
