registerAppResource

Registers your React UI as an MCP resource ChatGPT can render

Defined in widget.js. This function tells ChatGPT: "I have a UI at this URI — when you need it, here's the HTML to display, and here are the domains it's allowed to load scripts from."

What does registerAppResource do?

It registers your built React HTML as a fetchable MCP resource and configures CSP so the widget can load JS/CSS from your server.

Code Description 
// widget.js
registerAppResource(
  server,
  "horoscope-widget",
  WIDGET_URI,
  { _meta: { ui: { csp } } },
  async () => ({
    contents: [
      {
        uri: WIDGET_URI,
        mimeType: RESOURCE_MIME_TYPE,
        text: widgetHtml,
        _meta: { ui: { csp } },
      },
    ],
  })
);

Registers your built React HTML as a fetchable MCP resource and configures CSP so the widget can load JS/CSS from your server.

Argument What it means
server The MCP server instance — where this resource gets attached.
"horoscope-widget" Human-readable name shown in MCP resource lists.
WIDGET_URI Unique ID ChatGPT uses to request this UI (ui://widget/horoscope.html). See WIDGET_URI docs.
{ _meta: { ui: { csp } } } Security rules — which domains the widget is allowed to load JS/CSS from (your ngrok host).
async () => ({ contents: [...] }) Callback that runs when ChatGPT requests the resource. Returns the actual HTML.

What the callback returns

Field Description
uri Confirms which resource this is (WIDGET_URI).
mimeType text/html;profile=mcp-app — tells ChatGPT this is an MCP App widget, not plain HTML.
text The actual HTML from dist/index.html (your built React app, with asset URLs rewritten to your public server).
_meta.ui.csp CSP on the content item itself — takes precedence over the listing-level CSP.

Why does CSP appear twice?

CodeDescription
 
{ _meta: { ui: { csp } } }        // on the resource listing
_meta: { ui: { csp } }            // on the content item
  • First (listing): default CSP ChatGPT sees when browsing available resources.
  • Second (content): CSP ChatGPT actually applies when it fetches the HTML.

In this project both use the same csp object — they allow your ngrok domain to serve widget assets.

  3. ChatGPT calls resources/read for that URI
