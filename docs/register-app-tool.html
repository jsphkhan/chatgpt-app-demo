registerAppTool

Registers a tool ChatGPT can call — and links it to your UI

Defined in tools/show-horoscope.tool.js. This function exposes the show_horoscope capability to ChatGPT: what to call it, what inputs it needs, what UI to show, and what code runs when invoked.

What does registerAppTool do?

It registers an MCP tool that ChatGPT's model can call during a conversation, and attaches UI metadata so the horoscope card renders after the tool returns.

Code Description 
// tools/show-horoscope.tool.js
registerAppTool(
  server,
  "show_horoscope",
  {
    title: "Show horoscope",
    description:
      "Displays a personalized horoscope card based on the user's date of birth. " +
      "When the user asks for their horoscope, first ask for their date of birth (YYYY-MM-DD), " +
      "then call this tool with their DOB.",
    inputSchema: showHoroscopeInputSchema,
    outputSchema: showHoroscopeOutputSchema,
    _meta: {
      ui: {
        resourceUri: widgetUri,
        csp: widgetCsp,
      },
    },
  },
  handleShowHoroscope
);

Registers an MCP tool that ChatGPT's model can call during a conversation, and attaches UI metadata so the horoscope card renders after the tool returns.

Argument What it means
server The MCP server instance to attach this tool to.
"show_horoscope" Tool identifier — ChatGPT calls this name when invoking the tool.
title Short human-readable label for the tool.
description Instructions for the LLM. Tells ChatGPT when and how to use this tool — here, ask for DOB first, then call with YYYY-MM-DD.
inputSchema Zod schema defining the tool input — in this case dob (date of birth string).
outputSchema Zod schema describing the structured output returned to the UI (sign, mood, points, etc.).
_meta.ui.resourceUri Links this tool to the widget registered by registerAppResource. Must match WIDGET_URI.
_meta.ui.csp CSP domains allowed for the linked widget.
handleShowHoroscope Async handler — runs on your server when ChatGPT calls the tool. Returns text + structuredContent for the UI.

What the handler returns

CodeDescription
 
return {
  content: [
    { type: "text", text: `${horoscope.symbol} ${horoscope.sign}: ${horoscope.mood}` },
  ],
  structuredContent: horoscope,
};

The handler returns two payloads:

  • content — text reply ChatGPT shows in the conversation (e.g. "♌ Leo: Cosmic energy favors bold moves today.").
  • structuredContent — data for the React widget, read in App.tsx via app.ontoolresult → result.structuredContent.

Tool vs resource — how they work together

Description Diagram

registerAppTool defines what ChatGPT can do (call show_horoscope with a DOB) and points to which UI to render.

registerAppResource defines what UI to show — the actual React HTML at WIDGET_URI.

Step-by-step

Step What happens
1 User asks for their horoscope in ChatGPT
2 ChatGPT reads the description and asks for date of birth
3 ChatGPT calls show_horoscope with { dob: "1990-08-15" }
4 handleShowHoroscope runs on your server (with mock API delay)
5 Tool returns structuredContent → widget receives it via ontoolresult
6 Horoscope card renders in the chat

One-line summary

registerAppTool exposes a server action to ChatGPT, tells the model how to use it, links it to your widget UI, and runs your handler when called.
