src/App.tsx

React widget rendered inside ChatGPT's iframe

This file is the UI layer. It connects to ChatGPT via useApp, listens for tool events, and renders the horoscope card (or a loading skeleton).

What does useApp do?

Code Description 
const { isConnected, error } = useApp({
  appInfo: { name: "horoscope-widget", version: "0.1.0" },
  capabilities: {},
  onAppCreated: (app) => {
    app.ontoolinput = () => { ... };
    app.ontoolresult = (result) => { ... };
    app.ontoolcancelled = () => { ... };
  },
});

Creates an MCP App instance and connects it to ChatGPT over postMessage (your widget runs in an iframe; ChatGPT is the parent window).

Part What it means
useApp React hook that creates an App instance and connects it to ChatGPT via PostMessageTransport.
isConnected true once the MCP Apps handshake with ChatGPT completes. Show a connecting state until then.
error Set if the connection fails (timeout, handshake error, etc.).
appInfo Name and version of this widget — sent during initialization.
onAppCreated Callback to register event handlers before the connection finishes. This is where you wire up tool listeners.

How ChatGPT notifies your widget iframe

Your React app does not poll the MCP server or open a WebSocket. It sits inside an iframe embedded in ChatGPT. ChatGPT is the parent window; your widget talks to it over the browser postMessage API using the MCP Apps protocol (JSON-RPC messages).

Code Description 
// useApp creates this for you under the hood:
const transport = new PostMessageTransport(
  window.parent,  // send messages to ChatGPT
  window.parent   // only accept messages from ChatGPT
);
await app.connect(transport);

PostMessageTransport wraps window.parent.postMessage and a message event listener. Outgoing and incoming traffic is JSON-RPC — the same shape MCP uses elsewhere, but routed through the iframe boundary instead of HTTP.

1. Handshake (before any tool events)

Description Diagram

When the iframe loads, useApp connects and runs an initialization handshake:

  1. Widget sends ui/initialize (app name, version, capabilities)
  2. ChatGPT replies with host capabilities and context
  3. Widget sends ui/notifications/initialized
  4. isConnected becomes true

Handlers like ontoolresult must be registered in onAppCreated before this finishes, or early notifications can be missed.

        │  ui/initialize
        ▼
   ChatGPT (parent)
        │
        │  initialize response
        ▼
React widget
        │
        │  ui/notifications/initialized
        ▼
   isConnected = true

2. Tool lifecycle notifications (Host → View)

After ChatGPT calls show_horoscope on your MCP server, it pushes updates into the iframe. These are notifications (no response expected) with method names defined by the MCP Apps spec:

Notification Your handler When ChatGPT sends it
ui/notifications/tool-input app.ontoolinput Tool call has started; complete arguments are available (e.g. { dob: "1990-08-15" }). Widget shows skeleton.
ui/notifications/tool-result app.ontoolresult MCP server returned the tool result. Payload is a standard CallToolResult — includes content and structuredContent.
ui/notifications/tool-cancelled app.ontoolcancelled Tool call was cancelled (user action, timeout, etc.). Widget should clear loading state.

3. What the raw message looks like

Code Description 
// ChatGPT → iframe via postMessage (simplified)
{
  "jsonrpc": "2.0",
  "method": "ui/notifications/tool-result",
  "params": {
    "content": [
      { "type": "text", "text": "♌ Leo: Cosmic energy..." }
    ],
    "structuredContent": {
      "dob": "1990-08-15",
      "sign": "Leo",
      "symbol": "♌",
      "mood": "Cosmic energy favors bold moves today.",
      "points": ["...", "...", "..."],
      "accentColor": "#FF9F43"
    }
  }
}

PostMessageTransport receives this on the message event, validates event.source === window.parent, parses the JSON-RPC payload, and routes it inside the App class.

useApp abstracts the transport — you only assign app.ontoolresult = (result) => { ... } and read result.structuredContent. That is the same data your server returned from handleShowHoroscope in tools/show-horoscope.tool.js.

4. End-to-end path for this project

Description Diagram
  1. ChatGPT calls show_horoscope over HTTP to your MCP server
  2. Server runs handleShowHoroscope and returns the result
  3. ChatGPT already loaded your widget HTML from registerAppResource
  4. ChatGPT posts tool-input then tool-result into the iframe
  5. App.tsx handlers update React state and re-render the card

The MCP HTTP call and the iframe postMessage notifications are two separate channels. The server never talks to the widget directly — ChatGPT is the bridge.

Event handlers

Code Description 
app.ontoolinput = () => {
  setLoading(true);
  setHoroscope(null);
};
 Fires when ChatGPT starts calling the tool. Show the skeleton and clear any previous horoscope. 
app.ontoolresult = (result) => {
  setLoading(false);
  const data = result.structuredContent;
  if (data?.sign && data.points?.length) {
    setHoroscope(data);
  }
};
 Fires when the tool returns. Read structuredContent from the server and store it in React state to render the card. 
app.ontoolcancelled = () => {
  setLoading(false);
};
 Fires if the tool call is cancelled. Hide the loading skeleton.

Other ways to do this

1. Vanilla JS (same SDK, no React)

CodeDescription
 
import { App, PostMessageTransport } from "@modelcontextprotocol/ext-apps";

const app = new App({
  appInfo: { name: "horoscope-widget", version: "0.1.0" },
});

app.ontoolresult = (result) => {
  const { sign, mood } = result.structuredContent;
  document.getElementById("sign").textContent = sign;
};

await app.connect(new PostMessageTransport(window.parent));
 Same MCP Apps protocol and ontoolresult events — just without the React hook. You manage the DOM yourself.

2. Raw postMessage (OpenAI quickstart style)

CodeDescription
 
// JSON-RPC over window.parent.postMessage
await rpcRequest("ui/initialize", { appInfo, protocolVersion });
rpcNotify("ui/notifications/initialized", {});

window.addEventListener("message", (event) => {
  if (event.data?.method === "ui/notifications/tool-result") {
    updateFromResponse(event.data.params);
  }
});
 Hand-rolled bridge with no SDK. More code, but shows exactly what useApp does under the hood.

3. Legacy ChatGPT Skybridge (older pattern)

CodeDescription
 
// ChatGPT-specific — not MCP Apps standard
const toolOutput = useOpenAiGlobal("toolOutput");
 Some older ChatGPT widgets read window.openai.toolOutput directly. This project uses the open MCP Apps standard via @modelcontextprotocol/ext-apps instead.
