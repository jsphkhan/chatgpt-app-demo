src/App.tsx

React widget rendered inside ChatGPT's iframe

This file is the UI layer. It connects to ChatGPT via useApp, listens for tool events, and renders the horoscope card (or a loading skeleton).

What does useApp do?

Code Description 
const { isConnected, error } = useApp({
  appInfo: { name: "horoscope-widget", version: "0.1.0" },
  capabilities: {},
  onAppCreated: (app) => {
    app.ontoolinput = () => { ... };
    app.ontoolresult = (result) => { ... };
    app.ontoolcancelled = () => { ... };
  },
});

Creates an MCP App instance and connects it to ChatGPT over postMessage (your widget runs in an iframe; ChatGPT is the parent window).

Part What it means
useApp React hook that creates an App instance and connects it to ChatGPT via PostMessageTransport.
isConnected true once the MCP Apps handshake with ChatGPT completes. Show a connecting state until then.
error Set if the connection fails (timeout, handshake error, etc.).
appInfo Name and version of this widget — sent during initialization.
onAppCreated Callback to register event handlers before the connection finishes. This is where you wire up tool listeners.

How data gets from tool → UI

Description Diagram
  1. User asks for horoscope in ChatGPT
  2. ChatGPT calls show_horoscope on your MCP server (server.js → tools/)
  3. Tool handler returns: structuredContent: { sign, mood, points, ... }
  4. ChatGPT receives that result
  5. ChatGPT notifies your widget iframe
  6. app.ontoolresult fires in the widget
  7. You read result.structuredContent and update React state 
┌─────────────┐     MCP tool call      ┌──────────────┐
│   ChatGPT   │ ─────────────────────► │  server.js   │
│   (host)    │                        │  + tools/    │
└─────────────┘                        └──────────────┘
       │                                        │
       │  postMessage notification              │ structuredContent
       │  (ontoolresult)                        │ in tool response
       ▼                                        │
┌─────────────┐ ◄──────────────────────────────┘
│  React UI   │   useApp maintains the bridge
│  (iframe)   │   ontoolresult receives the data
└─────────────┘

Event handlers

Code Description 
app.ontoolinput = () => {
  setLoading(true);
  setHoroscope(null);
};
 Fires when ChatGPT starts calling the tool. Show the skeleton and clear any previous horoscope. 
app.ontoolresult = (result) => {
  setLoading(false);
  const data = result.structuredContent;
  if (data?.sign && data.points?.length) {
    setHoroscope(data);
  }
};
 Fires when the tool returns. Read structuredContent from the server and store it in React state to render the card. 
app.ontoolcancelled = () => {
  setLoading(false);
};
 Fires if the tool call is cancelled. Hide the loading skeleton.

UI states

Condition What renders
error Connection error message
!isConnected "Connecting to ChatGPT…"
loading HoroscopeSkeleton — shimmer placeholders
!horoscope Empty state — prompt user to ask ChatGPT
horoscope set Full horoscope card with sign, mood, and insight points

Other ways to do this

1. Vanilla JS (same SDK, no React)

CodeDescription
 
import { App, PostMessageTransport } from "@modelcontextprotocol/ext-apps";

const app = new App({
  appInfo: { name: "horoscope-widget", version: "0.1.0" },
});

app.ontoolresult = (result) => {
  const { sign, mood } = result.structuredContent;
  document.getElementById("sign").textContent = sign;
};

await app.connect(new PostMessageTransport(window.parent));
 Same MCP Apps protocol and ontoolresult events — just without the React hook. You manage the DOM yourself.

2. Raw postMessage (OpenAI quickstart style)

CodeDescription
 
// JSON-RPC over window.parent.postMessage
await rpcRequest("ui/initialize", { appInfo, protocolVersion });
rpcNotify("ui/notifications/initialized", {});

window.addEventListener("message", (event) => {
  if (event.data?.method === "ui/notifications/tool-result") {
    updateFromResponse(event.data.params);
  }
});
 Hand-rolled bridge with no SDK. More code, but shows exactly what useApp does under the hood.

3. Legacy ChatGPT Skybridge (older pattern)

CodeDescription
 
// ChatGPT-specific — not MCP Apps standard
const toolOutput = useOpenAiGlobal("toolOutput");
 Some older ChatGPT widgets read window.openai.toolOutput directly. This project uses the open MCP Apps standard via @modelcontextprotocol/ext-apps instead.
